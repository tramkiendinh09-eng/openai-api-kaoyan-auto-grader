# 考研数学阅卷规范蒸馏版

本文件由项目中的 1994-2025 年考研数学考试分析、官方答案解析与典型错误说明蒸馏而成，作为自动阅卷系统的固定阅卷上下文。它不是某一道题的官方评分细则；具体题目仍必须优先依据该题的官方答案、考试分析、参考解析和给定评分标准。若某题没有明确评分细则，模型只能依据参考解析拆分“推导评分点”，不得伪装为官方标准。

## 1. 总原则

1. 阅卷目标是判断考生是否掌握题目考查的核心概念、基本方法、计算能力和逻辑表达能力，不是只比较最终答案。
2. 选择题按最终选项判分。若 OCR 对选项识别有歧义，应标记人工复核。
3. 填空题按最终结果判分。结果可接受数学等价形式，如 `1/(-12)` 与 `-1/12` 等价；但缺少必要条件、区间端点、常数项、符号方向时不得满分。
4. 解答题必须按步骤给分。最终答案正确但过程错误、缺关键推理或存在偶然正确风险，不能给满分。
5. 方法不同但数学逻辑严谨、条件检查完整、结果等价，应正常给分。
6. 过程有一处计算错误但后续思路可延续，应给此前正确步骤分，并说明错误影响范围。
7. 关键定义、定理条件、区域边界、矩阵变换性质、概率独立条件等若误用，应扣相应关键分。
8. 书写或 OCR 不清时，若影响关键公式、系数、结论、题号对应关系，不得强行定分，应降低置信度并标记人工复核。

## 2. 解答题严格阅卷口径（重点）

本节用于模拟“京区式严格阅卷”的尺度，但不能称为公开官方京区评分细则。公开管理规定只能确认：全国统一命题科目由省级教育招生考试机构组织专家组，依据教育部教育考试院评分参考拟定本地区评卷细则，进行试评、评卷教师培训、业务指导和争议仲裁；具体评分参考按国家秘密管理。因此本项目只能把公开流程、本地考试分析和参考解析蒸馏为严格口径。

1. 大题先看可见过程，再看最终结果。没有可见推导证据的最终答案，最多只能给结论分，不能推定其掌握中间步骤。
2. 对 10 分左右解答题，若只有最终答案或只有零散公式，一般不超过满分的 30%；若主线正确但跳过关键定理条件、区域/边界/分布/矩阵性质检查，一般不超过满分的 70%-80%。
3. 对证明题，必须有“条件确认 -> 定理或构造 -> 推理链 -> 结论闭合”。只写结论、只画图、只凭直观、只给特殊情形，不能高分。
4. 对多问大题，前一问错误若直接影响后一问，后一问只能给与错误无关的独立步骤分；不能因为后续形式看似完整而自动给高分。
5. 对计算型大题，建模/定限/公式选择错误属于主线错误，后续机械计算即使整齐也不能高分；单纯算术错误只扣影响范围内的分。
6. 对极值、最值、收敛、拐点、根的个数、参数范围等题，必须检查边界、端点、定义域、符号变化、充分必要性或存在唯一性。只求驻点、只令导数为零、只代入一个候选值，不得满分。
7. 对积分与多元积分题，积分区域、上下限、变量替换雅可比、方向/法向、收敛性是关键评分点；区域或上下限错，相关后续积分分大幅扣除。
8. 对线性代数题，相似/合同/正交变换、秩、特征值、特征向量、正定性、惯性指数等概念不能混用。概念主线错误时，即使得到相近标准形，也只能给少量计算分。
9. 对概率统计题，分布函数/密度/分布律、样本独立同分布、似然函数、条件分布、积分区域、统计量分布条件是关键评分点。没有写出正确似然函数、分布函数或关键条件，后续估计量或期望方差不能高分。
10. 对非常规解法，只有在每一步逻辑自洽、条件完整、结果等价时正常给分；若只是套用辅导书形式、算子形式、图形直观或未经说明的结论，应按缺证据扣分。
11. “会做但没写”不给分。学生卷面没有出现的关键评分点，不得由模型脑补。
12. 若参考材料没有逐步分值，必须把拆出的步骤标为“推导评分点”。推导评分点要保守，宁可把不确定项标记人工复核，也不要给宽松满分。

## 3. 选择题阅卷规则

1. 只看最终选项，选项与参考答案一致得满分，不一致得 0 分。
2. 若学生同时写多个选项、涂改无法辨认、题号和答案串错位，应标记人工复核。
3. 不因学生过程文字看似合理而给选择题部分分。
4. 对考试分析中常见的概念误判，例如把驻点等同极值点、误判拐点、混淆收敛与绝对收敛，只在解释扣分原因时引用，不改变客观题按选项判分原则。

## 4. 填空题阅卷规则

1. 结果正确且与参考答案等价，得满分。
2. 等价判断包括分数化简、符号等价、矩阵或向量等价表达、区间等价写法、常数形式等价。
3. 若答案缺少定义域、参数范围、积分常数、方向条件、端点开闭，且这些是题目答案本质部分，应扣分或不得满分。
4. 若 OCR 将多位数拆开、上下标丢失、根号/积分上下限不清，影响结果判断，应标记人工复核。

## 5. 解答题分步评分规则

1. 先识别该题主线：高数、线代、概率统计、综合证明或应用建模。
2. 依据参考解析拆出关键评分点：建模/设元、定理适用条件、核心变形、关键计算、结论表达。
3. 若给定材料中没有官方分值分配，只能把这些点标为“推导评分点”。
4. 每个评分点必须对应学生答案中的可见证据；不能凭学生“应该会”而给分。
5. 关键步骤缺失时，即使结果相同，也应扣关键分。
6. 只要方法合法，不强制学生使用参考答案同一路线。
7. 证明题重视逻辑链条：条件、构造、定理引用、结论闭合缺一不可。
8. 应用题和建模题重视变量含义、约束条件、目标函数、边界检查和实际单位；只求驻点不查边界不得满分。
9. 若学生过程出现“关键结论直接跳出”，必须在扣分点中写明缺少哪个推导评分点，例如：缺少存在唯一性证明、缺少端点讨论、缺少积分区域说明、缺少分布函数推导、缺少特征值为 0 的证明。
10. 若模型认为应给 80% 以上分数，必须在有效步骤中列出足以支撑高分的关键链条；列不出时自动降低分数或标记人工复核。

## 6. 考试分析中高频扣分点

1. 极限与连续：等价无穷小使用条件错误，泰勒展开阶数不足，洛必达法则条件不满足，忽略左右极限或参数范围。
2. 一元微分：把导数为零误认为必有极值，极值点、拐点、单调区间和凹凸性概念混淆。
3. 一元积分：换元或分部积分边界错误，广义积分收敛性未判断，定积分对称性使用条件不足。
4. 多元微积分：偏导、全微分、隐函数求导链式关系错误；二重积分区域画错、上下限颠倒、极坐标雅可比漏乘。
5. 曲线曲面积分：方向、法向、路径无关条件、格林/高斯/斯托克斯公式适用条件漏检。
6. 级数：绝对收敛和条件收敛混淆，比较判别同阶量估计错误，幂级数端点漏检。
7. 微分方程：通解、特解、初值条件代入错误，常数项遗漏。
8. 线性代数：相似与合同混淆，初等行变换与列变换使用目的不清，秩、特征值、正定性、二次型惯性指数判断错误。
9. 概率统计：独立与不相关混淆，条件概率、联合密度积分区域、边缘分布、期望方差公式误用，统计量分布条件漏检。
10. 证明题：只写结论不写构造，定理条件不满足，逻辑方向错误，把必要条件当充分条件。
11. 过程证据不足：考试分析多次指出，没有推导过程、没有证明关键结论、没有讨论必要情形、未利用前问台阶、书写不规范或逻辑不严密，都会导致扣分；自动阅卷必须按这些缺陷严扣。
12. 多问联动：第（I）问中的密度、区域、参数、矩阵、函数表达式等基础结果错误，会直接影响后续问得分；后续若只是沿用错误结果计算，不能按正确主线给高分。

## 7. 评分输出约束

1. 必须输出结构化 JSON。
2. 每题必须包含分数、扣分原因、有效步骤、错误/缺失步骤、依据来源、置信度和是否人工复核。
3. 非官方来源必须标明为 `provided_reference` 或 `inferred_from_reference`。
4. 由参考解析拆出的步骤必须在文字中显式出现“推导评分点”。
5. 若题目、答案或学生作答的 OCR 质量不足以可靠判分，最终得分可为 `null`，并必须标记 `needs_human_review=true`。
6. 不得为缺失的官方评分细则编造分值分配。可以给暂定分，但必须说明依据是“推导评分点”且置信度较低。
7. 大题输出中必须明确写出“高分依据”或“封顶原因”。若存在关键步骤缺失、推理跳步、边界未检、条件未说明、前问错误影响后问，应在 `main_deducted_points` 或 `wrong_or_missing_steps` 中体现。
